# Editorial

Используй этот skill, когда нужно отредактировать, вычитать или улучшить технический текст на русском языке.

## Что делать

1. Сначала прочитай `./references/editorial-guide.md`.
2. Затем полностью прочитай целевой документ от начала до конца.
3. Выполни полный редакторский проход:
   - грамотность
   - стилистика
   - англицизмы и локализация терминов
   - типографика
   - Markdown-разметка
4. Если редактируешь Markdown-файл, используй `remark <FILE> --quiet --output` для правки синтаксиса Markdown вместо ручной косметики моделью.
5. После правок перечитай результат целиком и проверь, что терминология внутри текста остаётся консистентной.

## Правила применения

- Считай `./references/editorial-guide.md` каноническим набором правил для этого skill.
- Если в проекте есть более строгие локальные инструкции по языку или терминам, следуй им поверх общего гайда.
- Если пользователь просит не только замечания, а именно редактирование, выполняй правки end-to-end, а не останавливайся на рецензии.
