# Prompt

Используй этот skill, когда нужно написать, переработать, оценить или систематизировать промпт, системную инструкцию, агентный контракт или prompt-based workflow.

## Что делать

1. Сначала оцени, какой именно класс задачи перед тобой:
   - короткий прикладной промпт
   - системная инструкция
   - tool-calling / structured output
   - многошаговый агентный workflow
   - аудит существующего промпта
2. Затем открой только релевантные разделы `./references/prompt-anthology.md`, а не читай весь файл без необходимости.
3. На основе релевантных паттернов:
   - улучши структуру промпта
   - убери двусмысленность
   - задай чёткий формат выхода
   - отдели роль, правила, входные данные и ограничения
   - проверь, не конфликтуют ли инструкции между собой
4. Если задача исследовательская или архитектурная, явно формулируй компромиссы и границы применимости выбранного prompt-паттерна.

## Правила применения

- Считай `./references/prompt-anthology.md` каноническим справочником паттернов для этого skill.
- Для простых задач извлекай только нужный паттерн, не тащи в итоговый промпт всю антологию.
- Если проект уже использует устойчивый стиль системных инструкций, адаптируй рекомендации под него, а не переписывай всё под абстрактный идеал.
